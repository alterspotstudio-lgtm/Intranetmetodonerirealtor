# 08 · Registro y evidencias

Este documento sirve como base para preparar el sistema como producto registrable y defendible. No sustituye asesoría legal.

## Capas de protección recomendadas

### Marca

Preparar expediente de marca para:

- Método NERI.
- Variantes comerciales.
- Logotipo y estilo visual si aplica.

### Software

Preparar expediente del programa de computadora:

- Código fuente.
- Manual técnico.
- Manual de usuario.
- Capturas de pantalla.
- Arquitectura.
- Bitácora de versiones.
- Evidencia de autoría y fechas.

### Manuales y metodología

Preparar documentos separados:

- Manual operativo del asesor.
- Manual de experiencia del cliente.
- Manual de fases y candados.
- Manual de implementación para inmobiliarias.
- Manual comercial.

### Secreto empresarial

Mantener fuera del repo público:

- credenciales,
- webhooks completos,
- claves de Make,
- fórmulas privadas,
- contratos con proveedores,
- precios internos no publicados,
- documentación sensible de clientes.

## Evidencias que deben guardarse

- Fecha de creación de repo.
- Commits importantes.
- Capturas de prueba viva.
- Videos de funcionamiento.
- Export de estructura Airtable sin datos sensibles.
- Mapa de escenarios Make sin exponer tokens.
- Manual de usuario.
- Manual técnico.
- Bitácora de cambios.
- Contratos de cesión o confidencialidad con colaboradores.

## Carpeta privada sugerida fuera de GitHub público

`Metodo NERI / Registro /`

- `01 Marca`
- `02 Codigo fuente`
- `03 Manuales`
- `04 Evidencias de uso`
- `05 Capturas y videos`
- `06 Contratos colaboradores`
- `07 Bitacora versiones`
- `08 Auditorias`

## Regla de evidencia

Toda mejora importante debe quedar con:

- fecha,
- responsable,
- problema resuelto,
- archivo afectado,
- prueba realizada,
- captura o video,
- riesgo si se modifica.

## Antes de registrar

Revisar con especialista:

- titularidad del código,
- uso de librerías,
- nombre comercial,
- cesión de derechos de colaboradores,
- alcance del registro,
- documentación pública contra documentación privada.
