# NO ROMPER

Checklist de seguridad documental para Método NERI.

## Antes de cambiar algo

1. Crear rama nueva.
2. Leer el archivo vivo.
3. Revisar quién consume ese archivo.
4. Revisar campos conectados.
5. Probar con un caso real.
6. Documentar resultado.

## No hacer

- No cambiar folios sin auditoría.
- No cambiar campos de Airtable sin búsqueda completa.
- No enviar links incompletos al cliente.
- No mezclar vista vendedor con vista comprador.
- No publicar secretos.
- No vender pendientes como terminados.
