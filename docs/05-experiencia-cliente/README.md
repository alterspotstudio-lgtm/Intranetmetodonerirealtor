# 05 · Experiencia del cliente

## Principio

El cliente debe sentirse acompañado e informado. No debe saber ni escuchar palabras técnicas internas. La experiencia pública debe parecer simple, clara y profesional.

## Cliente vendedor

El vendedor debe vivir:

1. Confirmación de que su propiedad ya entró al proceso.
2. Portal de Venta con estado claro.
3. Acciones para avanzar.
4. Expediente documental para subir documentos.
5. Confirmación de producción inmobiliaria.
6. Confirmación de oferta, promesa y notaría cuando aplique.
7. Cierre formal de la venta.

## Cliente comprador

El comprador debe vivir:

1. Atención desde el interés en la propiedad.
2. Confirmación de cita.
3. Seguimiento claro si aparta.
4. Portal de avance con vista distinta al vendedor.
5. Solicitud documental cuando corresponda.
6. Confirmación de promesa, notaría y entrega.

## Mensajes públicos permitidos

- “Tu propiedad ya está en proceso.”
- “Este es el siguiente paso.”
- “Desde aquí puedes subir tus documentos.”
- “Tu asesor confirmará la disponibilidad final.”
- “Tu operación avanza a la siguiente etapa.”
- “Este link se mantiene actualizado.”

## Palabras que no deben usarse con cliente

- Intranet.
- CRM.
- Amazon.
- Make.
- Airtable.
- Webhook.
- Token.
- Base de datos.
- Automatización interna.
- Backend.
- API.

## Canales visibles

- WhatsApp.
- Portal de Venta.
- Expediente documental.
- Link de confirmación.
- Documento imprimible o PDF.

## Regla de claridad

Cada mensaje al cliente debe responder:

1. Qué pasó.
2. Qué sigue.
3. Qué tiene que hacer el cliente.
4. Qué hará el asesor.
5. Dónde puede verlo o confirmarlo.
